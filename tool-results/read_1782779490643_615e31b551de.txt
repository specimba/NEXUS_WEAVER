     1→# NEXUS Visual Weaver — Project Worklog
     2→
     3→## Project Overview
     4→Recreating the HuggingFace Space `build-small-hackathon/NEXUS_Visual_Weaver` (a governed
     5→multi-agent visual creation pipeline) inside the Next.js sandbox, with REAL working AI
     6→capabilities powered by the z-ai-web-dev-sdk (Image Generation, VLM, LLM skills).
     7→
     8→### Pipeline (the "golden path")
     9→1. Text Prompt Input — user describes the desired image (+ style / aspect / wardrobe options)
    10→2. FLUX.2 Image Generation — `zai.images.generations.create`  (replaces FLUX.2 Klein)
    11→3. ST3GG Security Scan — `zai.chat.completions.create` evaluates safety of prompt + image metadata
    12→4. MiniCPM-V Visual Judge — `zai.chat.completions.createVision` analyzes the generated image
    13→5. Nemotron Evidence Parse — `zai.chat.completions.create` produces structured JSON verdict
    14→6. Structured Output — persisted to SQLite via Prisma + shown to user
    15→
    16→### App Views (single `/` route, view-switcher)
    17→- **Studio** — interactive image generation (the main user surface)
    18→- **Command Center** — dashboard: metrics, status, recent activity, architecture layers
    19→- **Pipeline** — stage-by-stage flow visualization + timings
    20→- **Compliance** — safety/governance logs
    21→- **Gallery** — archive of generated images
    22→- **Monitor** — system / GPU / HF quota monitor
    23→
    24→### Tech
    25→- Next.js 16 App Router, TypeScript, Tailwind v4, shadcn/ui (New York)
    26→- Prisma + SQLite for persistence
    27→- z-ai-web-dev-sdk (backend only) for Image Gen / VLM / LLM
    28→- Zustand for client state, TanStack Query for server state
    29→- Framer Motion for transitions
    30→- Color system: emerald/teal + amber accents on near-black (NO indigo/blue)
